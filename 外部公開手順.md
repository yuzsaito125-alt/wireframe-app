# Render.com で公開する手順

## 全体の流れ
```
ローカルのファイル → GitHubリポジトリ → Render.com が自動デプロイ
```

---

## ステップ1：GitHubにリポジトリを作る

1. https://github.com にログイン（アカウントなければ無料で作成）
2. 右上の「+」→「New repository」
3. 設定：
   - Repository name: `wireframe-app`（任意）
   - Public / Private: どちらでもOK
   - 「Create repository」をクリック

---

## ステップ2：ローカルからGitHubにアップロード

コマンドプロンプトで以下を順番に実行：

```cmd
cd /d "e:\@YuzuruSaito\仕事フォルダ\@kiro\wireframe-app"

git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/あなたのユーザー名/wireframe-app.git
git push -u origin main
```

※ `あなたのユーザー名` は自分のGitHubユーザー名に置き換える
※ 途中でGitHubのログインを求められたらブラウザで認証する

---

## ステップ3：Render.com でデプロイ

1. https://render.com にアクセス → 「Get Started for Free」
2. GitHubアカウントでサインアップ（連携が楽）
3. ダッシュボードで「New +」→「Web Service」
4. 「Connect a repository」でさっき作った `wireframe-app` を選択
5. 設定画面：
   - **Name**: wireframe-app（任意）
   - **Region**: Singapore（日本から近い）
   - **Branch**: main
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app`
   - **Instance Type**: Free（無料）
6. 「Create Web Service」をクリック

---

## ステップ4：デプロイ完了を待つ

- 初回は3〜5分かかります
- ログが流れて最後に `==> Your service is live 🎉` と出ればOK
- URLは `https://wireframe-app-xxxx.onrender.com` のような形で固定されます

---

## ステップ5：QRコードでスマホからアクセス

1. ブラウザで `https://wireframe-app-xxxx.onrender.com` を開く
2. 「📱 QR」ボタンを押してQRコードを表示
3. スマホのカメラでスキャン
4. HTTPSなので音もカメラも使えます ✅

## スリープについて

無料プランは15分アクセスがないとスリープします。
復帰方法：URLにアクセスするだけ。30〜60秒で自動起動します。
（スリープ防止したい場合は有料プラン $7/月 へアップグレード）
